02) MY part -  MAINTENANCE & TICKETS Role structure 
Good question 🔥 — this is exactly how you should think for a real system.

---

# 🎯 SHORT ANSWER

👉 YES ✅
**Both Lecturer and Student can create tickets**

👉 They are BOTH:

```text
USER
```

---

# 🧠 ROLE STRUCTURE (CLEAR)

Your system should have:

```text
USER
  ├── Student
  └── Lecturer

TECHNICIAN
ADMIN (optional)
```

---

# 💡 IMPORTANT IDEA

👉 Lecturer ≠ separate role
👉 Student ≠ separate role

✔ They are **sub-types of USER**

---

# 🧩 HOW IT WORKS

## 👤 USER (Student + Lecturer)

👉 Can:

* Create tickets
* View their tickets
* Add comments

---

## 👨‍🔧 TECHNICIAN

👉 Can:

* View all tickets
* Assign tickets
* Update status
* Add comments

---

# 🎓 REAL EXAMPLES

## 👩‍🎓 Student

* “Chair is broken in lab”
* “WiFi not working”

---

## 👨‍🏫 Lecturer

* “Projector not working”
* “AC not functioning in lecture hall”

👉 Both create tickets the SAME way

---

# 🧠 HOW TO DESIGN THIS (IMPORTANT)

## OPTION 1 (BEST & SIMPLE)

### User table:

```text
id
name
email
password
role        → USER / TECHNICIAN
userType    → STUDENT / LECTURER
```

---

## WHY THIS IS GOOD

✔ Simple
✔ Flexible
✔ Matches real system

---

